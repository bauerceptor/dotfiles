# Architecture

Understanding how these dotfiles are organized and how they work.

## рџ“ђ Design Principles

1. **Platform Agnostic** - No hardcoded paths, works across systems
2. **Distro Agnostic** - Package manager abstraction for portability
3. **Modular** - Each tool in its own directory
4. **Stow-Based** - Uses GNU Stow for symlink management
5. **Automated** - Scripts for installation and updates
6. **Flexible** - Per-tool customization supported

## рџ”— How Stow Works

### The Stow Structure

Each "package" follows this structure:
```
package-name/
в””в”Ђв”Ђ .config/
    в””в”Ђв”Ђ tool/
        в””в”Ђв”Ђ config-file
```

When you run `stow package-name`, it creates a symlink:
```
~/.config/tool/config-file в†’ ~/.dotfiles/package-name/.config/tool/config-file
```

### Example: Fish Shell

```
fish/
в””в”Ђв”Ђ .config/
    в””в”Ђв”Ђ fish/
        в””в”Ђв”Ђ config.fish

After stowing:
~/.config/fish/config.fish в†’ ~/.dotfiles/fish/.config/fish/config.fish
```

## рџЋЁ Theme System Architecture

### Components

1. **Central Palettes** (`themes/schemes/*.toml`)
   - Define colors once in a standard format
   - TOML format for easy editing
   - Includes metadata (name, variant, description)

2. **Converters** (`themes/converters/*.py`)
   - Python scripts that read palettes
   - Generate tool-specific configurations
   - Each tool has its own converter

3. **Generated Configs** (`themes/generated/`)
   - Auto-generated by converters
   - Never edit these files directly
   - Recreated when themes are applied

4. **Symlinks**
   - Link generated configs to the right locations
   - `~/.config/alacritty/colors.toml` в†’ generated file
   - `~/.config/ghostty/theme-active` в†’ generated file

### Theme Application Flow

```
User runs: ./scripts/switch-theme.sh sakura-dark alacritty

1. Read palette: themes/schemes/sakura-dark.toml
2. Run converter: themes/converters/to_alacritty.py
3. Generate config: themes/generated/alacritty-sakura-dark.toml
4. Create symlink: ~/.config/alacritty/colors.toml в†’ generated file
5. User restarts terminal to see changes
```

### Per-Tool Independence

Each tool's theme is managed separately:
- Alacritty can use `sakura-dark`
- Ghostty can use `nord`
- Each tool has its own symlink
- No coupling between tools

## рџ”§ Platform Agnostic Design

### Environment Variables

All paths use environment variables:
- `$HOME` instead of hardcoded paths like `/home/username`
- `$XDG_CONFIG_HOME` for config paths (when available)
- Shell paths resolved from `$PATH`

### Package Manager Abstraction

The system detects available package managers:

```bash
# bash/.bashrc.d/package-manager.sh
if command -v dnf; then
    alias pkgi='sudo dnf install'
elif command -v apt; then
    alias pkgi='sudo apt install'
elif command -v pacman; then
    alias pkgi='sudo pacman -S'
fi
```

Universal aliases work everywhere:
- `pkgi` - install package
- `pkgs` - search packages
- `pkgu` - update system
- `pkgr` - remove package

### Tool Detection

Configs use flexible detection:

```bash
# Example: Homebrew detection
if command -v brew; then
    eval "$(brew shellenv)"
elif [ -d "$HOME/.linuxbrew" ]; then
    eval "($HOME/.linuxbrew/bin/brew shellenv)"
elif [ -d /opt/homebrew ]; then
    eval "(/opt/homebrew/bin/brew shellenv)"
fi
```

## рџ”„ Update Flow

```
User runs: ./scripts/update.sh

1. Git pull latest changes
2. Re-stow all packages (update symlinks)
3. Update Fish plugins (fisher update)
4. Optionally update fonts
5. Done!
```

## рџљЂ Installation Flow

```
User runs: ./scripts/bootstrap.sh

1. Detect Linux distribution
2. Detect package manager (dnf/apt/pacman)
3. Install core dependencies (git, stow, curl, wget)
4. Install modern CLI tools (eza, bat, fd, rg, etc.)
5. Install Nerd Fonts
6. Offer to set Fish as default shell

Then: ./scripts/install.sh

1. Stow all packages (create symlinks)
2. Install Fish plugins via Fisher
3. Done!
```

## рџ“¦ Adding New Tools

### 1. Create Package Directory

```bash
mkdir -p ~/.dotfiles/newtool/.config/newtool
```

### 2. Add Config Files

```bash
cp ~/.config/newtool/config ~/.dotfiles/newtool/.config/newtool/
```

### 3. Update Install Script

Edit `scripts/install.sh` and add to PACKAGES array:
```bash
PACKAGES=(
    ...
    "newtool"
)
```

### 4. Stow It

```bash
cd ~/.dotfiles
stow newtool
```

## рџ”ђ Security Considerations

### What's Ignored

`.gitignore` excludes:
- Fish variables (user-specific)
- LazyVim generated files
- OS files (.DS_Store, Thumbs.db)
- Editor temp files (*.swp, *~)
- Secrets (.env, *.key, *.pem)
- Local overrides (*.local)

### Never Commit

- API keys or tokens
- SSH keys
- Password files
- Any file with credentials

## рџ§Є Testing Changes

Before committing:

```bash
# Check dependencies
./scripts/check-deps.sh

# Test stow (without making changes)
stow -n -v newtool

# Test unstow
stow -D newtool

# Re-stow
stow newtool
```
